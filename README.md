# React SPA для просмотра коллекций MET Museum
#### выполнено в рамках тестового задания

**Met Explorer** — это одностраничное приложение (SPA) на **React + TypeScript + Redux Toolkit**,  
которое работает с открытым API крупнейшего музея мира —  
**The Metropolitan Museum of Art (MET)**, Нью-Йорк.

## Приложение позволяет:
### Можно украсть и не возвращать
У каждого экспоната есть две кнопочки - "лайкнуть" и "украсть". Возможности украсть [**нет на оригинальном сайте**](https://www.metmuseum.org/art/collection). Мы не передаем информацию никому. Пользуйтесь!

- просматривать отделы (departments) музея;
- автоматически выбирать случайный отдел при загрузке страницы;
- получать статистику по количеству экспонатов в выбранном отделе;
- выводить список доступных экспонатов для указанного департамента;

Проект создан как пример взаимодействия с внешним REST API  
и построения отзывчивого SPA на современном стеке.

# Метрополитен-музей (The MET Museum)

Приложение использует официальное открытое API музея:

https://metmuseum.github.io/

Это API предоставляет доступ к:
- 20+ отделам коллекции (Egyptian Art, European Paintings, Arms & Armor, Photography…)
- более чем **470 000** экспонатов
- полным карточкам объектов:  
  название, год, автор, страна, маленькое и большое изображения

## Функциональность приложения

### Загрузка списка отделов (departments)
При запуске приложение автоматически запрашивает полный список отделов музея.

### Случайный выбор отдела
После загрузки отделов приложение выбирает **случайный  департамент**.

### Поиск и загрузка экспонатов по отделу
Отдел загружается через endpoint:
```
/search?departmentId=ID&hasImages=true&q=*
```

API возвращает:
- `total` — количество экспонатов в отделе;
- `objectIDs[]` — список всех ID экспонатов.

### Отображение количества экспонатов
В хедере показывается:
```
Экспонатов найдено: 17 438
```
Автоматически обновляется при смене отдела.

### Выбор отдела вручную
В хедере есть комбобокс, позволяющий переключать департамент.  
При выборе запускается новый поиск.

### Маршрутизация

Роутинг построен на createHashRouter,
что обеспечивает корректную работу на GitHub Pages.

## Как запустить и всё такое

### Требуется Node.js версии 20.19.5
Установка через nvm:
```
nvm install 20.19.5
nvm use
```
### Установка
```bash
npm install
```
### Запуск
```
npm run dev

```
### Сборка
```
npm run build
```
### Запуск Линтера
```
npm run lint
```
или 
```
npm run lint:fix
```
### Исправить ошибки линтера
```
npx prettier --write .
```
### Загрузить на GitHub Pages
```
npm run deploy
```
